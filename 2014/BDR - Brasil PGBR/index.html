BDR

O que o BDR ?

  • Replicação bidirecion
    • Até 48 nós de mestre-mestre.
  • Recomendado para replicar entre áreas geográficas distante
  • Asynchronous
  • Eventualmente consistente
  • DDL é replicado de forma transparente
  • Licencia PGDG

Status atual do projeto

  • Ultima versão do RPM:0.7.1

¿ Quando precisamos BDR?

  • Nós geograficamente distribuídos.
  • Bancos de dados parcialmente distribuídos

¿ Como funciona ?

  • Logical decoding
  • Event triggers
  • Background workers
  • DDL deparse for Event Triggers (upcoming in 9.4)
  • Sequence AM - distributed
  • Replication Identifiers, Wal messages, Commit timestamps

Que es UDR?

  • Replicação unidirecional
  • Não requer patches. Apenas 9,4 com uma extensão.
  • Use decodificação lógica
  • Algumas limitações: Não Multi mestre, sem replicação DDL, as sequências não distribuídos.
  • Suporta replicação cruzada versão

Detalhes

  • Código de aplicação controlado e limpo.
  • Als odes seqüências devem ser definidos como globais para a aplicação.

DDL transparente

  • Exceções
    • No CREATE/ALTER/DROP DATABASE/TABLESPACE/USER
    • ALTER TABLE … ALTER COLUMN … USING();
    • ALTER COLUMN … ADD COLUMN … DEFAULT;
